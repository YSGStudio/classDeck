# 수업 도우미(가칭: ClassDeck) 서비스 계획서 — 브라우저 폴더 저장판

> 교사가 날짜별로 수업을 설계하고, 준비한 자료를 발표용으로 크게 띄워 수업을 진행한 뒤 간단히 회고하는 웹 서비스
> **회원가입 없이, 일반 웹사이트로 접속하며, 방문자가 자기 컴퓨터의 폴더를 직접 지정해 그 폴더에 데이터를 저장·불러옵니다.**

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명(가칭) | ClassDeck |
| 한 줄 소개 | "수업 준비부터 발표, 회고까지 — 내 PC 폴더에 그대로 저장" |
| 접속 방식 | 일반 웹사이트(호스팅) 접속, 별도 설치·서버 실행 불필요 |
| 데이터 저장 | 방문자가 **자기 컴퓨터에서 지정한 폴더**에 JSON·자료 파일로 저장 (브라우저 File System Access API) |
| 인증 | **없음** (데이터는 각자 자기 PC 폴더에 보관되므로 로그인 불필요) |
| 주요 사용자 | 초·중·고 교사 (1인 사용) |
| 브라우저 요건 | **크로미움 계열(Chrome·Edge·Opera 등)** — Safari·Firefox 미지원 |

### 1.1 핵심 동작
- 사이트 접속 후 **"폴더 선택"으로 내 컴퓨터의 폴더를 한 번 지정**
- 수업을 만들고 편집하면 그 폴더에 파일로 바로 저장됨
- 재방문 시 같은 폴더 접근을 한 번 허용하면 **저장된 수업이 자동으로 표시**됨
- 폴더를 통째로 복사하면 백업·이동 완료

### 1.2 저장 방식 설계 결정 (File System Access API)
- 방문자가 `showDirectoryPicker()`로 자기 PC의 폴더를 직접 선택 → 앱이 그 폴더에 파일을 읽고 씀
- 선택한 폴더 핸들을 브라우저 IndexedDB에 보관 → 재방문 시 **다시 선택할 필요 없이 한 번의 권한 허용 클릭**으로 자동 로드
- 서버에 데이터를 보내지 않으므로 별도 백엔드·DB·회원가입 불필요
- **제약**: File System Access API는 크로미움 계열 브라우저 + HTTPS 환경에서만 동작. 미지원 브라우저는 안내 메시지 표시(대안: 폴더 대신 파일 내보내기/가져오기 제공 고려)

## 2. 핵심 기능 명세

### 2.1 수업 관리 (날짜별 등록/조회)
- 날짜를 선택해 새 수업 생성 (제목·과목·학년 등 메타 입력)
- 지정 폴더의 수업 파일을 읽어 목록/캘린더로 표시
- 수업 클릭 시 준비모드로 진입
- 기존 수업 복제(다음 차시 재활용), 삭제

### 2.2 준비모드 (편집 화면)
교사가 아래 항목을 입력·저장:

| 순서 | 항목 | 형태 | 설명 |
|------|------|------|------|
| 1 | 성취기준 | 텍스트 | 교육과정 성취기준 |
| 2 | **탐구질문** | 텍스트 | 성취기준에서 이어지는 핵심 탐구질문 |
| 3 | 채점기준 | 표(상/중/하) | 평가 루브릭 |
| 4 | 도달목표 | 텍스트 | 이 수업에서 도달할 목표 |
| 5 | 수행과제 | 텍스트 | 학생 수행 과제 |
| 6 | 활동 섹션 (도입/활동/마무리) | 제목+내용+섹션별 자료+**타이머(분)** | 종류별로 자유롭게 추가, **드래그로 순서 변경**. 섹션마다 링크·이미지 자료 첨부 가능, **목표 시간(분)을 설정하면 발표모드에서 카운트다운 표시** (신규) |

저장 시 해당 수업의 JSON 파일이 지정 폴더에 갱신됩니다. (이전 버전의 "수업 자료" 전체 입력란은 폐지되었고, 자료는 활동 섹션마다 첨부합니다.)

### 2.3 자료 저장 (PPT / 링크 / 이미지)
- 자료는 수업 전체 단위가 아니라 **활동 섹션별로** 첨부 (도입에 영상 링크, 활동에 워크시트 이미지 등 섹션마다 따로)
- **링크**: URL을 JSON에 저장 → 발표모드에서 **"새 창에서 열기" 또는 "QR로 보기"** 선택 (신규, 2.8 참고)
- **이미지/PPT/문서**: 업로드 시 지정 폴더 하위 `materials/{수업ID}/activities/{섹션ID}/`에 실제 파일로 저장, JSON에는 상대 경로 기록
- 자료마다 제목·정렬 순서 지정

### 2.4 채점기준 (루브릭)
- 한 수업에 여러 채점기준 항목 추가
- 항목별 **상 / 중 / 하** 기준 텍스트 입력
- 발표모드에서도 표 형태로 노출

예시:
| 채점기준 | 상 | 중 | 하 |
|----------|----|----|----|
| 자료 분석력 | 근거를 들어 분석 | 부분적으로 분석 | 분석 미흡 |

### 2.5 발표모드 (프레젠테이션 화면)
- 준비모드 데이터를 **큰 글씨/슬라이드형**으로 렌더링
- 섹션 단위 이동(탐구질문 → 도달목표 → 수행과제 → 채점기준 → 활동(도입/활동/마무리) → 자료)
- 전체화면(Fullscreen API), 키보드 화살표 이동
- 편집 UI 숨김 → 학생에게 깔끔한 화면
- 활동 섹션에 타이머가 설정되어 있으면 큰 카운트다운과 시작/일시정지/리셋 버튼 표시 (신규, 2.7 참고)
- 활동 섹션의 링크 자료는 "QR로 보기"를 누르면 큰 QR코드로 전환되어 학생들이 직접 스캔해 참여 가능 (신규, 2.8 참고)

### 2.6 수업 후 피드백 (회고)
- 수업 후 간단한 자기 피드백(자유 텍스트) 입력
- 해당 수업 JSON에 함께 저장, 다음 수업 준비 시 참고

### 2.7 활동 타이머 (신규)
- 각 활동 섹션(도입/활동/마무리)에 **목표 시간(분)**을 선택적으로 설정
- 발표모드에서 해당 섹션 슬라이드를 보는 동안 큰 글씨의 카운트다운이 표시됨
- 시작 / 일시정지 / 리셋 컨트롤 제공, 0초가 되면 화면이 강조 색으로 바뀌어 시각적으로 알림
- 타이머 진행 상태는 저장하지 않음(매번 발표 시작 시 초기화) — 설정한 목표 시간(분)만 수업 JSON에 저장

### 2.8 학생 참여 QR 코드 (신규)
- 활동 섹션에 첨부한 **링크 자료**를 발표모드에서 QR코드로 전환해 화면에 크게 띄울 수 있음
- 학생은 자기 휴대폰 카메라로 QR을 스캔해 외부 참여 사이트(구글 설문, 패들렛, 멘티미터 등)에 바로 접속
- QR 생성은 **브라우저 안에서 즉시 계산**(클라이언트 사이드 라이브러리 사용) — 외부 QR 생성 서버를 호출하지 않으므로 "서버에 데이터를 보내지 않는다"는 원칙 유지
- 별도 백엔드·실시간 동기화 없음: QR은 이미 존재하는 외부 링크를 "스캔하기 쉽게 보여주는" 용도이며, 학생 응답을 이 앱이 수집하지는 않음

## 4. 화면(페이지) 구성

| 경로 | 화면 | 설명 |
|------|------|------|
| `/` | 대시보드 | 폴더 권한 허용 후 캘린더 + 수업 목록 자동 표시 |
| `/lessons/new` | 수업 생성 | 날짜·메타 입력 |
| `/lessons/[id]` | 준비모드 | 항목·채점기준·자료 편집/저장 |
| `/lessons/[id]/present` | 발표모드 | 프레젠테이션 렌더링 |
| `/settings` | 설정 | 데이터 폴더 다시 선택/변경, 브라우저 호환 안내 |

> 인증 페이지(로그인/회원가입)는 없습니다.

## 5. 데이터 저장 구조 (방문자 지정 폴더)

### 5.1 폴더 구조
방문자가 선택한 폴더(예: `D:/수업자료`) 안에:

```
지정폴더/
├─ lessons/
│   ├─ 2026-06-16_과학_빛의굴절.json
│   ├─ 2026-06-17_수학_분수.json
│   └─ ...
└─ materials/
    ├─ 2026-06-16_과학_빛의굴절/
    │   └─ activities/
    │       ├─ activity-1/
    │       │   └─ diagram.png
    │       └─ activity-2/
    │           └─ slide.pptx
    └─ ...
```

- 수업 하나 = JSON 파일 하나 (DB 불필요)
- 자료 파일은 수업 ID 폴더 아래, **활동 섹션 ID별 하위 폴더**에 실제 파일로 저장
- 목록은 `lessons/` 폴더의 파일들을 읽어 구성

### 5.2 수업 JSON 스키마

```json
{
  "id": "2026-06-16_과학_빛의굴절",
  "lessonDate": "2026-06-16",
  "title": "빛의 굴절",
  "subject": "과학",
  "grade": "6학년",
  "achievementStandard": "빛이 다른 물질을 통과할 때 굴절함을 설명할 수 있다.",
  "inquiryQuestion": "빛은 왜 물속에서 꺾여 보일까?",
  "rubrics": [
    {
      "criteria": "현상 설명",
      "high": "원리까지 정확히 설명",
      "mid": "현상은 설명하나 원리 부족",
      "low": "설명 미흡"
    }
  ],
  "goal": "빛의 굴절 현상을 실험으로 관찰하고 설명한다.",
  "task": "물컵 실험 결과를 그림과 글로 정리하기",
  "activities": [
    {
      "id": "activity-1",
      "orderNo": 1,
      "kind": "intro",
      "title": "도입",
      "content": "물컵에 담긴 빨대 관찰",
      "durationMinutes": 5,
      "materials": [
        { "type": "image", "title": "굴절 도해", "path": "materials/2026-06-16_과학_빛의굴절/activities/activity-1/diagram.png" }
      ]
    },
    {
      "id": "activity-2",
      "orderNo": 2,
      "kind": "activity",
      "title": "굴절 실험",
      "content": "굴절 실험 수행",
      "durationMinutes": 15,
      "materials": [
        { "type": "link", "title": "실험 결과 제출 폼", "url": "https://forms.gle/..." }
      ]
    },
    {
      "id": "activity-3",
      "orderNo": 3,
      "kind": "closing",
      "title": "정리",
      "content": "결과 발표 및 정리",
      "durationMinutes": 5,
      "materials": []
    }
  ],
  "materials": [],
  "feedback": "활동2 시간이 부족했음. 다음엔 도입 단축.",
  "createdAt": "2026-06-16T08:00:00",
  "updatedAt": "2026-06-16T09:10:00"
}
```

> `inquiryQuestion`(탐구질문)이 `achievementStandard`(성취기준) 바로 다음에 추가되었습니다.
>
> **활동(activities) 스키마 변경**: 각 활동은 고유 `id`, 종류 `kind`(`intro`/`activity`/`closing`), 자기만의 `materials` 배열, 그리고 발표모드 타이머용 `durationMinutes`(분, 선택)를 가집니다. 최상위 `materials` 배열은 과거 버전과의 호환을 위해 남겨둔 필드로, 새 수업에서는 비워둔 채 사용하지 않습니다(자료는 항상 활동 섹션에 첨부).

### 5.3 폴더 선택·기억 방식
- 최초: `/settings` 또는 대시보드에서 "폴더 선택" → `showDirectoryPicker()`로 폴더 지정
- 선택한 디렉터리 핸들을 **IndexedDB에 저장** → 재방문 시 자동으로 불러와 권한 재확인(한 번 클릭) 후 로드
- 폴더가 비어 있으면 최초 진입 시 `lessons/`, `materials/` 하위 폴더를 생성

## 6. 기술 아키텍처

| 영역 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | Next.js (App Router) | 정적/일반 호스팅 가능 (Vercel 등) |
| UI | React, Tailwind CSS(권장) | 발표모드 대형 타이포 |
| 데이터 저장 | **브라우저 File System Access API** | 방문자 지정 폴더에 직접 읽기/쓰기 |
| 폴더 핸들 보관 | IndexedDB | 재방문 시 폴더 자동 인식 |
| 인증/백엔드 | 없음 | 서버에 데이터 전송하지 않음 |
| QR 코드 생성 | 클라이언트 사이드 QR 라이브러리 (예: `qrcode`) | 외부 QR 생성 서버 호출 없이 브라우저에서 즉시 생성 (신규) |
| 타이머 | React state + `setInterval` | 저장하지 않는 화면 전용 상태, 새로고침/재진입 시 초기화 (신규) |
| 배포 | Vercel 등 일반 호스팅 (HTTPS 필수) | API는 File System Access 동작 조건 |

### 6.1 주요 동작 (브라우저 측, 서버리스)
| 동작 | 구현 |
|------|------|
| 폴더 선택 | `const dir = await window.showDirectoryPicker()` |
| 권한 확인 | `dir.queryPermission()` / `dir.requestPermission()` |
| 수업 목록 로드 | `dir`의 `lessons/` 순회하며 각 `.json` 읽기 (`getFileHandle` → `getFile` → `text()`) |
| 수업 저장 | `getFileHandle(name,{create:true})` → `createWritable()` → `write(JSON)` |
| 자료 업로드 | `materials/{id}/activities/{섹션ID}/` 하위에 파일 핸들 생성 후 바이너리 write |
| 자료 표시 | 파일 핸들에서 `getFile()` → `URL.createObjectURL()`로 이미지·PPT 표시/다운로드 (새 창은 `noopener` 없이 열어야 blob URL이 정상 로드됨) |
| QR 표시 (신규) | 링크 자료의 URL을 QR 라이브러리로 인코딩 → `<canvas>`/`<svg>`로 화면에 렌더, 서버 호출 없음 |
| 타이머 (신규) | 섹션 진입 시 `durationMinutes * 60`초에서 `setInterval`로 1초씩 감소, 0이 되면 강조 스타일 적용 |
| 폴더 기억 | 선택한 디렉터리 핸들을 IndexedDB에 저장/복원 |

### 6.2 자동 불러오기 흐름
1. 사이트 접속
2. IndexedDB에 저장된 폴더 핸들 확인
   - 있으면: 권한 재확인 클릭 → `lessons/` 읽어 자동 표시
   - 없으면: "폴더 선택" 안내 → 선택 후 로드
3. 화면에 수업 목록/캘린더 표시

### 6.3 브라우저 호환 처리
- File System Access API 미지원 브라우저(Safari·Firefox) 감지 시 안내 메시지 노출
- (선택 대안) 미지원 환경에서는 수업 데이터를 **JSON/ZIP 파일로 내보내기·가져오기**하는 보조 기능 제공 검토

### 6.4 발표모드 구현 포인트
- 같은 수업 JSON을 받아 편집 UI 없이 큰 글씨/슬라이드형 렌더
- Fullscreen API + 키보드(←/→) 네비게이션
- 섹션 배열(탐구질문 → 도달목표 → 수행과제 → 채점기준 → 활동 → 자료)을 한 화면당 하나씩 표시
- 활동 섹션에 `durationMinutes`가 있으면 카운트다운 타이머를 함께 표시 (신규)
- 활동 섹션의 링크 자료는 "새 창에서 열기" / "QR로 보기" 중 선택해 표시 (신규)

## 8. 운영·백업 메모
- **백업**: 지정 폴더를 복사하면 끝. 클라우드 동기화 폴더(드라이브/원드라이브)를 지정하면 자동 백업·다기기 사용
- **프라이버시**: 데이터가 서버로 전송되지 않고 방문자 PC 폴더에만 저장됨
- **주의**: 데이터가 그 폴더에만 있으므로 폴더 삭제 시 복구 불가 → 주기적 백업 권장
- **브라우저**: 크로미움 계열 권장. 다른 브라우저는 기능 제한 안내 필요
