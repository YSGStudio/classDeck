# 수업 도우미(가칭: ClassDeck) 서비스 계획서 — 브라우저 폴더 저장판

> 교사가 날짜별로 수업을 설계하고, 준비한 자료를 발표용으로 크게 띄워 수업을 진행한 뒤 간단히 회고하는 웹 서비스
> **회원가입 없이, 일반 웹사이트로 접속하며, 방문자가 자기 컴퓨터의 폴더를 직접 지정해 그 폴더에 데이터를 저장·불러옵니다.**

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명(가칭) | ClassDeck |
| 한 줄 소개 | "수업 준비부터 발표, 회고까지 — 내 PC 폴더에 그대로 저장" |
| 접속 방식 | 일반 웹사이트(호스팅) 접속, 별도 설치·서버 실행 불필요 |
| 데이터 저장 | 방문자가 **자기 컴퓨터에서 지정한 폴더**에 JSON·자료 파일로 저장 (브라우저 File System Access API) |
| 인증 | **없음** (데이터는 각자 자기 PC 폴더에 보관되므로 로그인 불필요) |
| 주요 사용자 | 초·중·고 교사 (1인 사용) |
| 브라우저 요건 | **크로미움 계열(Chrome·Edge·Opera 등)** — Safari·Firefox 미지원 |

### 1.1 핵심 동작
- 사이트 접속 후 **"폴더 선택"으로 내 컴퓨터의 폴더를 한 번 지정**
- 수업을 만들고 편집하면 그 폴더에 파일로 바로 저장됨
- 재방문 시 같은 폴더 접근을 한 번 허용하면 **저장된 수업이 자동으로 표시**됨
- 폴더를 통째로 복사하면 백업·이동 완료

### 1.2 저장 방식 설계 결정 (File System Access API)
- 방문자가 `showDirectoryPicker()`로 자기 PC의 폴더를 직접 선택 → 앱이 그 폴더에 파일을 읽고 씀
- 선택한 폴더 핸들을 브라우저 IndexedDB에 보관 → 재방문 시 **다시 선택할 필요 없이 한 번의 권한 허용 클릭**으로 자동 로드
- 서버에 데이터를 보내지 않으므로 별도 백엔드·DB·회원가입 불필요
- **제약**: File System Access API는 크로미움 계열 브라우저 + HTTPS 환경에서만 동작. 미지원 브라우저는 안내 메시지 표시

## 2. 핵심 기능 명세

### 2.1 수업 관리 (날짜별 등록/조회)
- 대시보드: 캘린더(수업 있는 날짜에 점 표시) + 수업 목록, 제목·성취기준으로 검색
- 날짜를 선택해 새 수업 생성 (날짜·제목·과목 드롭다운·학년 드롭다운 입력)
- 수업 카드에서 **제목·날짜를 바로 인라인 수정** 가능 — 날짜를 바꾸면 캘린더의 점 표시도 즉시 이동 (수업 식별자/파일명은 생성 시점 값으로 유지, 내용만 갱신)
- 기존 수업 복제(다음 차시 재활용), 오늘 날짜로 복사, 삭제

### 2.2 준비모드 (편집 화면)
교사가 아래 항목을 순서대로 입력·저장 (발표모드도 같은 순서로 진행):

| 순서 | 항목 | 형태 | 설명 |
|------|------|------|------|
| 1 | 성취기준 | 텍스트 | 교육과정 성취기준 (성취기준 탭에서 자동 입력 가능) |
| 2 | 탐구질문 | 텍스트 | 성취기준에서 이어지는 핵심 탐구질문 |
| 3 | 도달목표 | 텍스트 | 이 수업에서 도달할 목표 |
| 4 | 수행과제 | 텍스트 | 학생 수행 과제 |
| 5 | 채점기준 | 기준 + 상/중/하 각 1줄 | 평가 루브릭, 여러 항목 추가 가능 |
| 6 | 활동 (도입/활동/마무리) | 제목 + 내용 + 분 타이머 + 수업도구 | 종류별로 자유롭게 추가, 드래그로 순서 변경. "활동"만 활동1·활동2처럼 번호가 붙음 |
| 7 | 수업 자료 | 링크/이미지/파일 | 수업 전체에서 공통으로 쓰는 자료(PPT, 이미지, 링크 등) |
| 8 | 수업 후 피드백 | 텍스트 | 자유 기술 |

저장은 우측 상단/하단 "저장" 버튼을 눌러야 폴더의 JSON 파일에 반영됩니다 (대시보드 카드의 제목·날짜 인라인 수정만 예외적으로 즉시 저장).

### 2.3 활동(도입/활동/마무리)과 수업도구
- 활동은 종류(도입/활동/마무리)를 선택해 자유롭게 추가하고 드래그로 순서 변경
- 활동마다 **목표 시간(분)**을 선택적으로 설정 — 발표모드에서 큰 카운트다운으로 표시(2.6 참고)
- 활동마다 **수업도구**(링크 전용) 등록 공간을 별도로 둠 — 수업 전체의 "수업 자료"와는 분리된, 그 활동에서만 쓰는 참여형 링크(구글 설문, 패들렛, 멘티미터 등)
- 발표모드에서 수업도구 링크는 자료 카드 형태로 표시되며, 클릭하면 **새 창에서 열기**, "QR로 보기"를 누르면 **큰 QR코드**로 학생이 스캔해 바로 참여할 수 있음 (2.7 참고)

### 2.4 수업 자료 (PPT / 링크 / 이미지)
- 활동별이 아니라 **수업 전체 단위**로 한 곳에 등록 (활동마다 따로 첨부하던 이전 방식은 폐지)
- **링크**: URL을 JSON에 저장 → 발표모드에서 클릭 시 모달로 미리보기(이미지/웹페이지) 또는 새 창, "QR로 보기"로 전환 가능
  - 유튜브 워치/공유 링크는 그대로는 iframe에서 재생이 막히므로, 자동으로 임베드 가능한 `/embed/` 형식 URL로 변환해서 재생되도록 처리
  - iframe으로 열리지 않는 사이트를 위해 미리보기 모달에 항상 "새 창에서 열기" 버튼 제공
- **이미지**: 업로드 시 지정 폴더 하위 `materials/{수업ID}/`에 실제 파일로 저장, 발표모드에서 모달로 크게 미리보기
- **PDF**: 업로드 시 같은 방식으로 저장하되, 발표모드에서는 모달이 아니라 화면 대부분을 차지하는 **인라인 PDF 뷰어**로 임베드(코너 위젯을 가리지 않도록 우측 여백 확보)
- **기타 파일**(pptx, docx 등): 미리보기 없이 새 창으로 최선 노출
- 자료마다 제목 지정, 추가한 순서대로 노출

### 2.5 채점기준 (루브릭)
- 한 수업에 여러 채점기준 항목 추가
- 항목별 **기준 / 상 / 중 / 하**를 각각 한 줄씩 입력 (3열 표가 아니라 세로로 나열 — 긴 설명도 잘리지 않게)
- 발표모드에서도 항목당 카드 한 장에 기준 1줄 + 상/중/하 각 1줄, 총 4줄로 표시

예시:
| 채점기준 | 상 | 중 | 하 |
|----------|----|----|----|
| 자료 분석력 | 근거를 들어 분석 | 부분적으로 분석 | 분석 미흡 |

### 2.6 활동 타이머
- 활동 슬라이드에는 항상 카운트다운 타이머가 떠 있음(준비모드에서 분을 지정하지 않으면 기본 5분, 발표 중에도 분 단위로 +/- 조정 가능)
- 시작 / 일시정지 / 리셋 컨트롤 제공, 0초가 되면 위젯이 강조 색으로 바뀌어 시각적으로 알림
- 타이머 진행 상태는 저장하지 않음(매번 발표 시작 시 초기화) — 설정한 목표 시간(분)만 수업 JSON에 저장

### 2.7 학생 참여 QR 코드 & 발표뽑기
- **QR 코드**: 수업 자료 또는 활동의 수업도구에 등록한 **링크**를 발표모드에서 QR코드로 전환해 화면에 크게 띄울 수 있음. 학생은 휴대폰 카메라로 스캔해 외부 참여 사이트에 바로 접속
  - QR 생성은 브라우저 안에서 즉시 계산(클라이언트 사이드 `qrcode` 라이브러리) — 외부 QR 생성 서버를 호출하지 않음
  - 별도 백엔드·실시간 동기화 없음: 이미 존재하는 외부 링크를 스캔하기 쉽게 보여주는 용도이며, 학생 응답을 이 앱이 수집하지는 않음
- **발표뽑기**: 설정에서 등록한 학생 명단(번호·이름·성별)을 바탕으로, 발표모드 코너 위젯에서 성별(랜덤/남/여)과 인원수를 선택해 무작위로 뽑기. 결과는 화면 중앙 모달에 이름 카드로 표시되며 아무 곳이나 클릭하면 닫힘
- 타이머와 발표뽑기 위젯은 화면 우측 상단에 항상 같은 크기로 나란히 표시됨

### 2.8 발표모드 (프레젠테이션 화면)
- 화면은 단 두 가지만 존재: **준비모드(편집 화면)**와 **전체화면 발표**. 그 중간 단계(전체화면이 아닌 발표 화면)는 없음
- "발표모드"를 클릭하는 즉시 전체화면으로 진입 후 바로 발표 화면이 뜸. Esc·F11·브라우저 UI 등 어떤 방식으로든 전체화면이 풀리면 즉시 준비모드로 돌아감
- 슬라이드 순서: 제목 → 탐구질문 → 도달목표 → 수행과제 → 채점기준 → 활동(도입/활동/마무리, 등록한 순서대로) → 수업 자료
- 화면을 좌우로 드래그(마우스/터치 공통)해 다음/이전 슬라이드로 이동. 별도의 이전/다음 버튼이나 쪽수 표시는 없음(상단의 얇은 진행률 바로만 위치 확인)
- 콘텐츠 영역은 코너 위젯 자리만 비우고 화면 거의 전체 폭을 사용하며, 글자 크기를 크게 잡아 줄바꿈을 최소화
- 수업 자료는 화면 우측 하단 "수업자료" 버튼을 눌러야 나오는 목록에서 선택해 열어봄
- 학생에게는 편집용 UI가 전혀 보이지 않는 깔끔한 화면

### 2.9 수업 후 피드백 (회고)
- 수업 후 간단한 자기 피드백(자유 텍스트) 입력
- 해당 수업 JSON에 함께 저장, 다음 수업 준비 시 참고

### 2.10 성취기준 탭
- 대시보드 옆에 **"성취기준"** 탭 — 2022 개정 교육과정 성취기준을 학년군·교과별로 탐색
- **데이터 출처**: 별도 데이터베이스 없이, CSV 자료를 빌드 시 JSON으로 변환해 앱 코드에 직접 포함(정적 데이터) → 폴더 연결과 무관하게 항상 동일한 내용을 바로 조회 가능
- 화면 구성
  - 학년군 탭(전체 / 3~4학년군 / 5~6학년군), 교과별 카드(영역 수·성취기준 수 표시) → 교과 클릭 시 해당 교과의 영역별 성취기준 목록으로 드릴다운
  - 성취기준, 코드, 영역명으로 즉시 검색 가능(검색 중에는 교과 구분 없이 결과를 바로 나열)
  - 각 성취기준 항목을 펼치면 성취수준(상/중/하)과 해설을 확인할 수 있음
- **"수업작성하기" 버튼**: 클릭한 성취기준 텍스트를 임시 보관한 뒤 "새 수업 만들기" 화면으로 이동 → 날짜·제목·과목·학년을 입력하고 만들기를 누르면 보관해둔 성취기준이 자동으로 채워짐 (같은 성취기준으로 같은 날짜·과목·제목이 이미 있으면 생성을 막고 제목을 바꿔달라고 안내)

## 3. 사용자 플로우

```
[사이트 접속]
     │
     ▼
[폴더 선택/권한 허용] ──▶ 지정 폴더 자동 로드 ──▶ [대시보드: 캘린더/목록/검색]
     │                                              │
     │(날짜 선택 또는 성취기준에서 "수업작성하기")        │(기존 수업 클릭 또는 카드에서 바로 수정)
     ▼                                              ▼
[새 수업 생성] ───────────────────────────▶ [준비모드: 입력/저장 → 폴더에 파일 기록]
                                                    │
                                                    ▼
                                    [발표모드: 클릭 즉시 전체화면 진입]
                                                    │
                                          (전체화면 종료 → 준비모드로 자동 복귀)
```

## 4. 화면(페이지) 구성

| 경로 | 화면 | 설명 |
|------|------|------|
| `/` | 대시보드 | 폴더 권한 허용 후 캘린더 + 수업 목록 자동 표시, 카드에서 제목·날짜 인라인 수정, 검색 |
| `/standards` | 성취기준 | 교과·학년군별 성취기준 탐색, 검색, "수업작성하기"로 바로 수업 생성 |
| `/lessons/new` | 수업 생성 | 날짜·제목·과목(드롭다운)·학년(드롭다운) 입력 |
| `/lessons/[id]` | 준비모드 | 항목·채점기준·활동(타이머/수업도구)·수업자료·피드백 편집/저장 |
| `/lessons/[id]/present` | 발표모드(전체화면) | 프레젠테이션 렌더링. 전체화면이 아닌 중간 상태는 존재하지 않음 |
| `/settings` | 설정 | 데이터 폴더 다시 선택/연결 해제, 학생 명단(번호·이름·성별) 등록, 브라우저 호환 안내 |

> 인증 페이지(로그인/회원가입)는 없습니다.

## 5. 데이터 저장 구조 (방문자 지정 폴더)

### 5.1 폴더 구조
방문자가 선택한 폴더(예: `D:/수업자료`) 안에:

```
지정폴더/
├─ lessons/
│   ├─ 2026-06-16_과학_빛의굴절.json
│   ├─ 2026-06-17_수학_분수.json
│   └─ ...
├─ materials/
│   ├─ 2026-06-16_과학_빛의굴절/
│   │   ├─ slide.pptx
│   │   └─ diagram.png
│   └─ ...
└─ students.json
```

- 수업 하나 = JSON 파일 하나 (DB 불필요)
- 자료 파일(이미지/PDF/기타 파일)은 **수업 ID 폴더 아래에 평면적으로** 저장 (활동별 하위 폴더 없음 — 자료는 수업 전체 단위로 등록하기 때문)
- 활동의 "수업도구"는 링크만 다루므로 파일을 저장하지 않고 JSON에 URL만 기록
- 학생 명단은 폴더 최상위 `students.json`에 저장
- 목록은 `lessons/` 폴더의 파일들을 읽어 구성

### 5.2 수업 JSON 스키마

```json
{
  "id": "2026-06-16_과학_빛의굴절",
  "lessonDate": "2026-06-16",
  "title": "빛의 굴절",
  "subject": "과학",
  "grade": "6학년",
  "achievementStandard": "빛이 다른 물질을 통과할 때 굴절함을 설명할 수 있다.",
  "inquiryQuestion": "빛은 왜 물속에서 꺾여 보일까?",
  "goal": "빛의 굴절 현상을 실험으로 관찰하고 설명한다.",
  "task": "물컵 실험 결과를 그림과 글로 정리하기",
  "rubrics": [
    {
      "criteria": "현상 설명",
      "high": "원리까지 정확히 설명",

    }

    {
      "id": "activity-1",
```

### 5.3 폴더 선택·기억 방식
- 최초: `/settings` 또는 대시보드에서 "폴더 선택" → `showDirectoryPicker()`로 폴더 지정
- 선택한 디렉터리 핸들을 **IndexedDB에 저장** → 재방문 시 자동으로 불러와 권한 재확인(한 번 클릭) 후 로드
- 폴더가 비어 있으면 최초 진입 시 `lessons/`, `materials/` 하위 폴더를 생성
- `/settings`에서 "폴더 연결 해제"로 기억된 폴더를 초기화할 수 있음

## 6. 기술 아키텍처

| 영역 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | Next.js (App Router, Turbopack) | 정적/일반 호스팅 가능 (Vercel 등) |
| UI | React, Tailwind CSS | 발표모드 대형 타이포 |
| 데이터 저장 | **브라우저 File System Access API** | 방문자 지정 폴더에 직접 읽기/쓰기 |
| 폴더 핸들 보관 | IndexedDB | 재방문 시 폴더 자동 인식 |
| 인증/백엔드 | 없음 | 서버에 데이터 전송하지 않음 |
| QR 코드 생성 | 클라이언트 사이드 `qrcode` 라이브러리 | 외부 QR 생성 서버 호출 없이 브라우저에서 즉시 생성 |
| 성취기준 데이터 | 빌드 타임에 포함되는 정적 JSON | CSV를 변환해 앱 코드에 내장 — DB·서버 호출 없음, 폴더 미연결 상태에서도 조회 가능 |
| 타이머 | React state + `setInterval` | 저장하지 않는 화면 전용 상태, 새로고침/재진입 시 초기화 |
| 전체화면 | Fullscreen API | 사용자 클릭(user-activation) 안에서 동기적으로 요청, 잃으면 즉시 준비모드로 복귀 |
| 슬라이드 이동 | Pointer Events 기반 좌우 드래그 | 마우스·터치 공통, 별도 버튼 없음 |
| 링크 임베드 보정 | `lib/embedUrl.ts` | 유튜브 watch/공유 URL을 `/embed/` 형식으로 변환해 iframe 재생 가능하게 처리 |
| 배포 | Vercel 등 일반 호스팅 (HTTPS 필수) | API는 File System Access 동작 조건 |

### 6.1 주요 동작 (브라우저 측, 서버리스)
| 동작 | 구현 |
|------|------|
| 폴더 선택 | `const dir = await window.showDirectoryPicker()` |
| 권한 확인 | `dir.queryPermission()` / `dir.requestPermission()` |
| 수업 목록 로드 | `dir`의 `lessons/` 순회하며 각 `.json` 읽기 (`getFileHandle` → `getFile` → `text()`) |
| 수업 저장 | `getFileHandle(name,{create:true})` → `createWritable()` → `write(JSON)` |
| 자료 업로드 | `materials/{수업ID}/` 하위에 파일 핸들 생성 후 바이너리 write |
| 자료 표시 | 파일 핸들에서 `getFile()` → `URL.createObjectURL()`로 이미지·PDF·기타 파일 표시/다운로드 |
| QR 표시 | 링크의 URL을 QR 라이브러리로 인코딩 → `<img>` 데이터 URL로 화면에 렌더, 서버 호출 없음 |
| 타이머 | 활동 슬라이드 진입 시 분 설정값(또는 기본 5분)에서 `setInterval`로 1초씩 감소, 0이 되면 강조 스타일 적용 |
| 전체화면 진입/복귀 | 클릭 핸들러 안에서 `requestFullscreen()` 동기 호출 후 클라이언트 사이드 라우팅 → `fullscreenchange` 이벤트를 단일 트리거로 모든 종료 경로를 감지해 준비모드로 복귀 |
| 폴더 기억 | 선택한 디렉터리 핸들을 IndexedDB에 저장/복원 |

### 6.2 자동 불러오기 흐름
1. 사이트 접속
2. IndexedDB에 저장된 폴더 핸들 확인
   - 있으면: 권한 재확인 클릭 → `lessons/` 읽어 자동 표시
   - 없으면: "폴더 선택" 안내 → 선택 후 로드
3. 화면에 수업 목록/캘린더 표시

### 6.3 브라우저 호환 처리
- File System Access API 미지원 브라우저(Safari·Firefox) 감지 시 안내 메시지 노출

### 6.4 발표모드 구현 포인트
- 같은 수업 JSON을 받아 편집 UI 없이 큰 글씨/슬라이드형 렌더, 코너 위젯 자리만 비우고 거의 전체 폭 사용
- 전체화면 상태와 화면 종류(준비모드 vs 발표모드)가 항상 1:1로 묶여 있도록 `fullscreenchange`를 단일 판단 기준으로 사용 — Esc·F11·브라우저 UI 등 어떤 경로로 전체화면이 풀려도 동일하게 준비모드로 복귀
- 좌우 드래그(Pointer Events)로 슬라이드 이동, 이전/다음 버튼·쪽수 표시 없음
- 활동 슬라이드에 항상 카운트다운 타이머와 발표뽑기 위젯(우측 상단, 같은 크기)이 함께 표시
- 활동의 수업도구·수업 자료의 링크는 "새 창에서 열기" / "QR로 보기" 중 선택해 표시, 유튜브 링크는 자동으로 임베드 형식으로 변환
- PDF 자료는 모달이 아니라 화면 대부분을 차지하는 인라인 뷰어로 표시

## 8. 운영·백업 메모
- **백업**: 지정 폴더를 복사하면 끝. 클라우드 동기화 폴더(드라이브/원드라이브)를 지정하면 자동 백업·다기기 사용
- **프라이버시**: 데이터가 서버로 전송되지 않고 방문자 PC 폴더에만 저장됨
- **주의**: 데이터가 그 폴더에만 있으므로 폴더 삭제 시 복구 불가 → 주기적 백업 권장
- **브라우저**: 크로미움 계열 권장. 다른 브라우저는 기능 제한 안내 필요
